# 🚀 SCRAPNINJA AWS DEPLOYMENT - QUICK START GUIDE

## ⚡ 30-Minute Fast Track Setup

### **Phase 1: Pre-Deployment (5 minutes)**

1. **Clone Repository & Update:**
```bash
cd c:\Users\nalin\Desktop\scapninja
git pull origin main
```

2. **Install Required Tools:**
   - AWS CLI: https://aws.amazon.com/cli/
   - Terraform: https://www.terraform.io/downloads
   - Docker Desktop: https://www.docker.com/products/docker-desktop

3. **Configure AWS:**
```powershell
aws configure
# Region: ap-south-1
# Enter your Access Key ID and Secret Key
```

---

### **Phase 2: Infrastructure Deployment (15 minutes)**

**On Windows (PowerShell):**
```powershell
cd deploy
.\deploy.ps1
```

**On Mac/Linux:**
```bash
cd deploy
chmod +x deploy.sh
./deploy.sh
```

This will:
- ✅ Create VPC with security groups
- ✅ Deploy RDS PostgreSQL (single-AZ, cost-optimized)
- ✅ Create SQS queue (replaces Kafka - 99% cost reduction)
- ✅ Setup EC2 auto-scaling (t3.small instances)
- ✅ Configure ALB (Application Load Balancer)
- ✅ Setup S3 + CloudFront CDN
- ✅ Create Route53 DNS records
- ✅ Setup CloudWatch monitoring

---

### **Phase 3: Application Deployment (10 minutes)**

**The deployment script will output:**
```
DEPLOYMENT_OUTPUTS.env
├─ ALB_DNS: [your-alb-dns]
├─ RDS_ENDPOINT: [your-rds-endpoint]
├─ SQS_QUEUE_URL: [your-sqs-url]
├─ S3_BUCKET: [your-s3-bucket]
└─ ACCOUNT_ID: 396287094911
```

1. **Get EC2 IPs:**
```bash
aws ec2 describe-instances --filters "Name=instance-state-name,Values=running" --query 'Reservations[*].Instances[*].{ID:InstanceId,PublicIP:PublicIpAddress}' --region ap-south-1
```

2. **SSH into instances and verify:**
```bash
ssh -i your-key.pem ec2-user@PUBLIC_IP
curl http://localhost:9000/health
```

3. **Deploy Frontend:**
```bash
cd frontend
npm run build
aws s3 sync out/ s3://scrapninja-static-396287094911/ --delete
```

4. **Update GoDaddy DNS:**
   - Go to GoDaddy DNS Management
   - Point A record to ALB DNS from DEPLOYMENT_OUTPUTS.env
   - Wait 15-30 minutes for propagation

## 🚨 Important Security Notes

1. **Database Password:**
   - Generated during terraform setup
   - Stored in terraform.tfvars (KEEP SECRET)
   - Never commit to git

2. **AWS Credentials:**
   - Use IAM user with minimal permissions
   - Enable MFA on AWS account
   - Rotate access keys monthly

3. **SSL Certificates:**
   - Auto-generated by AWS ACM (FREE)
   - Auto-renewed annually

4. **Backups:**
   - RDS: Automated daily (7-day retention)
   - S3: Versioning enabled
   - EBS: Snapshots (optional, costs extra)

## 🔄 Scaling Up Later

If you exceed budget due to traffic:

1. **Add Reserved Instances (40% discount):**
```bash
aws ec2 purchase-reserved-instances --reserved-instances-offering-id [OFFERING_ID]
```

2. **Add Multi-AZ for database (for HA):**
```bash
# Modify in terraform/main.tf: multi_az = true
terraform apply
```

3. **Use Spot Instances for non-critical services (70% off):**
```bash
# Modify in terraform/main.tf: use spot instances
terraform apply
```

## ❌ Scaling Down to Save Money

If under budget:

1. **Use 1 EC2 instead of 2:**
   - Terraform: `instance_count = 1`
   - Saves: $22/month

2. **Use db.t3.micro instead of small:**
   - Terraform: `db_instance_class = "db.t3.micro"`
   - Already optimized!

3. **Reduce backup retention:**
   - Terraform: `backup_retention_period = 3`
   - Saves: $10/month

## 🆘 Troubleshooting

### **Terraform fails to apply:**
```bash
terraform destroy -auto-approve  # Clean up
# Fix errors and reapply
terraform apply
```

### **EC2 instances not deploying:**
- Check user data logs: `ssh ... && tail -f /var/log/cloud-init-output.log`
- Verify security groups allow traffic
- Check IAM role permissions

### **Database connection failed:**
```bash
# Test from EC2
psql -h [RDS_ENDPOINT] -U admin -d appdb -c "SELECT 1"
```

### **SQS messages not processing:**
- Check Dead Letter Queue: `aws sqs receive-message --queue-url [DLQ_URL]`
- Verify IAM permissions
- Check application logs
